1. Overview

This backend exposes a banking assistant API that:

  • Authenticates clients with JWT (demo /auth/token; replace with corporate IdP in production).
  • Applies a dynamic gateway: default mode (JWT only) vs general mode (JWT + X-API-Key).
  • Routes natural language through Groq (primary) or OpenAI (fallback) using function calling, with a regex fallback if no LLM keys are set.
  • Executes money movement via the CBA mock server / UAPI (GetAccountsByCustomerId, LocalFundsTransfer), not in-memory mocks.
  • Maps chat payee names to NUBAN account numbers using CBA_PAYEE_ACCOUNT_MAP.

4. Request flows

4.1 Direct transfer (no LLM)

  1. POST /auth/token → access_token
  2. POST /api/v1/transfers with Authorization: Bearer … and body {"receiverUsername":"mahesh","amount":500}
  3. Server resolves mahesh → NUBAN via CBA_PAYEE_ACCOUNT_MAP, calls CBA LocalFundsTransfer

4.2 Chat transfer (LLM + CBA)

  1. Same JWT as above
  2. POST /api/v1/chat with {"message":"send 500 to mahesh"}
  3. Groq (or OpenAI) returns transfer_money tool args → same CBA path as 4.1

7. CBA / UAPI integration

  • Accounts: GET {CBA_BASE_URL}/api/Account/GetAccountsByCustomerId?customerId=…&authToken=…
  • Transfer: POST {CBA_BASE_URL}/thirdpartyapiservice/apiservice/CoreTransactions/LocalFundsTransfer with JSON body including FromAccountNumber, ToAccountNumber, Amount (string, major units), RetrievalReference, AuthenticationKey.
  • Payee names: resolved only through CBA_PAYEE_ACCOUNT_MAP; align keys with NLU output (lowercase).
  • Users: edit src/data/cbaUsers.js so customerId and accountNumber match your simulator database.

8. LLM orchestration

  • If GROQ_API_KEY is set: Groq OpenAI-compatible API + tools.
  • On Groq failure: OPENAI_API_KEY if set.
  • If neither key is set: regex orchestrator only.
  • Tools: transfer_money (receiver + amount), banking_knowledge_question (RAG placeholder).

9. Postman testing

  1. Create request POST http://localhost:<PORT>/auth/token, body JSON {"username":"alice","password":"alice123"}.
  2. Copy access_token.
  3. New request POST /api/v1/transfers → Authorization Bearer Token → body {"receiverUsername":"mahesh","amount":500}.
  4. If DEFAULT_GATEWAY_ENABLED=false, add header X-API-Key: <GENERAL_API_KEY>.

10. Security notes

  • Never commit .env or API keys; rotate keys if exposed.
  • Replace /auth/token with enterprise OAuth2/OIDC and JWKS validation for production.
  • Rate-limit and WAF belong in front of this service in production.
  • LLM output must not bypass server-side validation; CBA enforces real balances and account rules.
